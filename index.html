Programming Fundamentals

This introductory course teaches the fundamental concepts of using and programming computers through practical experience and problem-solving in a high-level language.

Students who successfully complete this paper should be able to:

  1. Create simple algorithms
  2. Express an algorithm as a program
  3. Write technical computer programs that incorporate fundamental programming constructs such as conditionals, repetition, functions, pointers and recursion
  4. Write technical computer programs that use console input/output and text files
  5. Design, implement, test and debug simple computer programs

Computer Science Fundamentals

An introduction to computer science: the discipline, computer systems, the theory of problem-solving and computer applications. Practical laboratory work is an important part of this paper.

Students who successfully complete this paper should be able to:

  1. Write technical computer programs relating to a range of topics including the use of simple data structures and command line operations
  2. Design simple logic circuits using Boolean algebra
  3. Describe the fundamental elements of a computer and the importance of the binary number system
  4. Design simple assembler programs for a virtual machine

Algorithms & Data Structures

Structured types. Array, list, tree and graph algorithms. Hash tables. Dynamic data structures. Abstract data types. Laboratory work is an important part of this course.

Students who successfully complete this paper should be able to:

  1. Represent lists, stacks, queues, and other data structures in an appropriate way for the target programming language.
  2. Explain the advantages/disadvantages of particular representations of these structures in particular circumstances.
  3. Design and implement computer programs that make use of these structures.
  4. Discuss the concepts of Abstract Data Types, including encapsulation and data hiding.

Object-Oriented Programming

Introduction to Object-Oriented programming; classes, objects, templates, inheritance, polymorphism, iterators, object libraries.

Students who successfully complete this paper should be able to:

  1. Design and implement C++ programs using object-oriented programming techniques.
  2. Design and implement C++ programs , using inheritance, polymorphism, exception handling.
  3. Design and implement programs, using templates and the STL object library.

Declarative Programming

Basic functional programming: functions, pattern matching, types, recursion, list processing. Basic logic programming: clauses, recursion, structures, arithmetic, list processing. Programming language concepts. Programming paradigms and language selection.

Students who successfully complete this paper should be able to:

  1. Design, write and run functional and logic programs using simple programming concepts: types, functions, recursion, pattern matching, facts, rules, predicates.
  2. Analyse the correctness and evaluate the design of given functional and logic programming modules.
  3. Design, write, test and run programs using advanced programming concepts: data abstraction, list, polymorphisms and overloading.
  4. Analyse a moderately complex given problem, identify and design data types and modules necessary to solve the problem using basic and advanced functional programming concepts, write, run and test the program.
  5. Analyse a moderately complex given problem, identify and design the required data types and modules necessary to solve the problem using basic and advanced logic programming concepts, write, run and test the program.
  6. Describe the main attributes of the declarative programming paradigm, the difference between functional and logic programming languages, and the difference between imperative and declarative programming paradigm.

Graphical Programming

Programming graphical-user-interfaces using an event-driven model. Drawing and transformation of graphical objects. Rendering in 3D. Animation techniques. Programming with graphical libraries.

Students who successfully complete this paper should be able to:

  1. Describe key concepts in computer graphics.
  2. Design computer programs that display simple graphics in 2D and 3D using Java.
  3. Discuss the graphics concepts pertaining to computer games and virtual reality simulations.
  4. Identify the advantages and disadvantages of various graphics systems and technologies.

Artificial Intelligence

AI programming. State space representation and search. Heuristics. Planning. Game playing. Knowledge representation. Knowledge-based systems. Natural language processing. Machine learning. Reasoning under uncertainty. Philosophical issues.

Students who successfully complete this paper should be able to:

  1. Describe the main Artificial Intelligence techniques used in computing.
  2. Design and program computers to store and manipulate knowledge.
  3. Describe, and in some cases program, the special techniques that can be used to perform intelligent tasks such as searching, game playing, natural language processing and machine learning by computers.
  4. Use logic for inference and fuzzy logic and some of its applications.
  5. Identify the advantages and disadvantages of applying various AI techniques in solving real world problems through exploring the various applications of AI.

Computer Networks

A layered approach to data communications and the Internet protocols.

Students who successfully complete this paper should be able to:

  1. Describe key computer network concepts, architectures and protocols.
  2. Relate algorithms, network services and products to each other.
  3. Apply concepts and principles to network design and management using the current standards.
  4. Apply fundamental network analysis tools.
  5. Discuss professional and ethical issues relating to data communication systems and their application, especially concerning Internet security.
  6. Apply the recent acquired knowledge to develop or apply simple network related programs.

Internet Programming

Advanced concepts of programming computers across the Internet: scripting, HTML, client and server-side programs, distributed objects, distributed object frameworks, security.

Students who successfully complete this paper should be able to:

  1. Describe the main techniques used in Internet Programming.
  2. Implement a database driven Internet Application using PHP and MySQL.
  3. Implement a server side application using servlets, JSPs and J2EE components.
  4. Apply the Internet Programming design concepts in solving real world problems.
  5. Identify the advantages and disadvantages of various Internet Programming techniques to real world problems.
  6. Demonstrate knowledge of new and emerging internet programming technologies.

Computer Application and the information age

A broad based and practical introduction to the applications of IT and computers that pervade today's society. A range of software applications will be studied in their context of use, enabling an understanding not only of technical issues but also the social, ethical and organisational aspects of applied computing.

Students who successfully complete this paper should be able to:

  1. Explain key components of stand-alone and networked information technology systems.
  2. Explain key concepts relating to software development and data storage.
  3. Discuss the impact of information technology on individuals, business and society.
  4. Apply software tools to the creation, analysis and presentation of data.
  5. Illustrate how computer applications are specified and built by applying a variety of tools.

Systems management and testing

A discussion of the technical concepts and software issues that IT managers require to plan, operate and maintain computer-based systems and software assets. The paper also considers some core issues in software testing.

Students who successfully complete this paper should be able to:

  1. Describe different software system contexts including open source and legacy software.
  2. Contrast the general features, benefits, and limitations of different approaches to software system acquisition, development and management.
  3. Identify the key aspects and decisions related to system testing, acceptance, and evaluation.
  4. Use appropriate tools and techniques for software testing and the management of software assets.
  5. Formulate software testing strategies.

Systems analysis and design

A study of methods used to capture, analyse and model system requirements. Students will acquire practical skills through case study work utilising techniques and software tools used by industry.

Students who successfully complete this paper should be able to:

  1. Describe the fundamental concepts and techniques of systems analysis.
  2. Apply the basic principles and techniques for eliciting, gathering and documenting the functional requirements of an IT system.
  3. Discuss and explain different software development life cycle models and issues concerned with the development of high quality software.
  4. Incrementally and iteratively use the Unified Modelling Language (UML) to model systems from requirements.
  5. Demonstrate practical skills in the use of at least one Computer Aided Software Engineering (CASE) tool.
  6. Apply object-oriented principles and architectures to the specification of software systems

Application Software and web development

An introduction to web-based application development. This paper explores a selection of topics and technologies relevant to the design and creation of web-based systems.

Students who successfully complete this paper should be able to:

  1. Analyse business and usage models for web-based technologies.
  2. Identify appropriate analysis and design techniques to develop web applications, including graphical design.
  3. Contrast the technology layers of web applications and be able to implement some of them.
  4. Create web sites containing textual and multimedia content.
  5. Select appropriate technologies to implement web-based applications that may include client side processing and various forms of structured and semi structured data.

User Experience Design

The course moves from conceptual analysis to the design, prototyping and implementation and evaluation of user interfaces. It emphasises development processes that result in interfaces that provide a superior user experience. Examples of real-life interfaces, from sources such as desktop, multimedia, web, novel, and mobile technologies are used. Significant practical work component.

Students who successfully complete this paper should be able to:

  1. Discuss the iterative nature of the user interface development lifecycle (IDLC), including user and task modelling.
  2. Define interface usability in the context of different kinds of applications, interaction devices, paradigms, metaphors and styles.
  3. Apply critical thinking and aesthetic judgement to the assessment of user interfaces.
  4. Use iterative design techniques to design and test a prototype for an elegant user interface.
  5. Carry out an interface evaluation.
  6. Work successfully in a group on projects related to interface design and evaluation.

Database Development

A study of the application of software engineering principles to the development of databases for information systems. Modern database principles, issues and development methods including conceptual modelling, logical and physical design, implementation, database languages (e.g. SQL), distributed database design, and methods for interfacing with the web are covered. Students will acquire practical skills utilising modern tools and techniques used by industry.

Students who successfully complete this paper should be able to:

  1. Implement a logical database design using SQL to create, populate and maintain it.
  2. Write queries to relational databases using SQL.
  3. Apply the rules of normalization to a relational database table.
  4. Understand the fundamentals of databases, including the concepts of transaction management and concurrency control, and in particular of the relational model of data.
  5. Create a conceptual database design using the ER model, from an understanding of user requirements.
  6. Understand the different operators of Relational Algebra.

Introductory Statistics

Applied statistics emphasising applications in the sciences and social sciences. Use of graphs and numbers to summarise and interpret data; data collection with surveys and experiments; elementary probability and sampling distributions to describe variability; inference for means, proportions, contingency tables and regression.

Students who successfully complete this paper should be able to:

  1. Choose graphical and numerical summaries of a batch of values and use them to answer questions about variability in the data.
  2. Summarise the relationship between two numerical or categorical variables.
  3. Critically examine data collected by others and the conclusions that they draw.
  4. Effectively present the information that you extract from data to others.
  5. Collect data with surveys and experiments to help answer real-life questions.
  6. Use distributions and sampling to model variability in data.
  7. Estimate population characteristics with confidence intervals. These characteristics include means, proportions, differences between means, differences between proportions, regression slopes and predictions from regression lines.
  8. Use hypothesis tests to assess whether there are differences between groups or relationships between variables.
